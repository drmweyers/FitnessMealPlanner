# EvoFit Meals — Business Vault Digital Product Specification

**Version:** 1.0.0
**Created:** 2026-04-21
**Based on:** EvoFit Trainer Business Vault Spec v3.0.0 (quality reference)
**Purpose:** Tiered digital product bundles included with each pricing tier. Hybrid delivery: highest-value items live in-app (zero friction), remaining items as branded PDFs via authenticated download.

---

## Design Principles

1. **Outcomes over items.** Every vault product is framed around the result it produces, not the thing itself.
2. **Consumption over collection.** In-app delivery for anything a trainer uses mid-session. PDFs only for reference material read once.
3. **Qualitative depth per tier.** Professional upgrades are better systems, not just more items. Enterprise upgrades are delegation-ready and team-scalable.
4. **Specificity sells.** Top 3 items carry specific, credible outcome claims.
5. **Nutrition-first framing.** Every item helps trainers build revenue through nutrition coaching and meal planning — not general business advice.

## Delivery Architecture

### In-App Products (Items #1-3)

| Product                   | Integration Point                                                                             | Implementation                                                                                                                 |
| ------------------------- | --------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| Meal Plan Templates       | Meal Plan Builder → "Vault Templates" section → one-click duplicate                           | DB: `vault_templates` with `category='meal-plan'`, `tier`, `content` (JSON meal plan structure). Rendered inline.              |
| Nutrition Email Templates | Client profile → "Templates" tab → category filter → one-click copy                           | DB: `vault_templates` with `category='email'`, `tier`, `subject`, `body` fields.                                               |
| Dietary Restriction Guide | Client profile → "Dietary Info" tab → select condition → see safe swaps + flagged ingredients | DB: `vault_templates` with `category='dietary'`, `tier`, `condition`, `safe_swaps[]`, `flagged_ingredients[]`. Interactive UI. |

### Downloadable Products (Items #4-14)

| Layer    | Technology                                        | Detail                                                              |
| -------- | ------------------------------------------------- | ------------------------------------------------------------------- |
| Storage  | DigitalOcean Spaces (existing) or local PDF serve | PDFs stored securely, not in `public/`.                             |
| Auth     | `GET /api/vault/download/:slug`                   | JWT required → tier check → serve PDF                               |
| Tracking | `vault_downloads` Drizzle table                   | userId, slug, tier, version, timestamp. Rate limited: 10/user/hour. |

### Fonts

Outfit and Inter loaded from Google Fonts in HTML templates. PDF rendering via Puppeteer `setContent()`.

## Quality Gates

1. **Trainer credibility** — Every product survives a nutrition-savvy trainer reading it critically.
2. **Outcome framing** — Top 3 items carry specific, credible outcome claims.
3. **Consumption-first** — In-app items have zero-friction delivery.
4. **Tier coherence** — Each tier is a strict superset. Professional is qualitatively deeper, not just more.
5. **No fabrications** — No feature claims without codebase backing.
6. **PDF quality** — >100KB, fonts rendered, backgrounds visible, no blank pages.
7. **Scope boundaries** — Nutrition scripts include clear scope-of-practice disclaimers where medical referral is appropriate.
